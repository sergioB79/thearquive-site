01 / Technical Reading

What exists today

Project type
A static sports research microsite presented as an intelligence brief.
Framework
The project is one authored index.html plus an OG preview image, with no build step, package manifest, or framework dependency.
Routing
There is no multi-page routing; the experience is split into tabbed panels for teams, matchups, bracket, and probabilities inside the same document.
Content model
The page combines editorial commentary, probability tables, visual comparisons, and bracket narratives, explicitly versioned as a verified snapshot.
Styling system
All styling is inline and custom, using IBM Plex and Bebas Neue to produce a newsroom-meets-scouting-report surface.
Deployment note
The Open Graph metadata points to GitHub Pages, so the repo is intended as a lightweight published artifact rather than an evolving application backend.

02 / Notes

Observed structure

It is essentially a match-study PDF mentality translated into a browser surface.

  • The technical base is minimal, but the information design is dense and deliberate.
  • The versioning language inside the page matters: it frames the work as a dated analytical snapshot.
  • This belongs in the archive as a research artifact, not as a general football site.
