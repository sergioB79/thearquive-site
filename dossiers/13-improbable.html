01 / Technical Reading

What exists today

Project type
A structurally mature publication site rather than a loose page collection: archive homepage, article routes, about route, analytics, social metadata and content pipeline all exist.
Framework
No formal framework. The current production-minded version uses static HTML, one large app.js client layer, and a Node generator script.
Routing
The live model supports /, /about, and real article URLs at /article/<slug>, while still behaving like one coherent client-rendered publication in the browser.
Content model
Articles are parsed from a single source file artigos1a10, paired with numbered images, dated in ARTICLE_DATES, and rendered into blocks such as headings, quotes and lists.
Static generation
generate-static-pages.js produces real shareable entry pages for every article and the About route so crawlers can read route-specific metadata.
Analytics
GoatCounter is integrated with manual pageview tracking because the public reading flow is client-routed.
Deployment
The active site is explicitly prepared for Vercel; vercel.json is minimal because the routing model is handled in the static structure itself.

02 / Notes

Observed structure

This is the only project in the folder that already behaves like a publication system rather than a page.

  • The current new/ version is clearly the authoritative branch, while older HTML drafts remain nearby as historical residue.
  • The README is unusually complete and describes operational maintenance, not just concept.
  • The site’s architecture is shaped by one real editorial constraint: social platforms need static metadata even when readers experience a JS publication app.
  • The large folder weight comes mostly from images and generated article pages, not from a framework toolchain.
  • The editorial worldbuilding is already deep enough to create its own publication identity.
