01 / Technical Reading

What exists today

Project type
A local automation and orchestration tool, not a standalone image model repository.
Framework
Python command-line app built with Typer and Rich, plus a tiny FastAPI server for serving generated images.
Routing
Minimal local API surface in server.py: root health endpoint and static image serving under /images.
Generation backend
Communicates with AUTOMATIC1111 over the local Stable Diffusion API at http://127.0.0.1:7860 by default.
Automation flows
Supports direct CLI generation, a watcher polling Reflect for 🎨 [[SD: ...]] triggers, and manual/hotkey-triggered generation.
Publishing path
Generated images can be saved locally, exposed via the local image server, and uploaded to Google Drive before appending a link into Reflect.
Config model
Runtime settings are stored outside the project in ~/.sdcli/config.json.

02 / Notes

Observed structure

This is a bridge layer: not the model itself, but the workflow around prompting, image retrieval and note-driven automation.

  • main.py defines the main CLI surface: generate, list models, switch models, samplers, config and likely other operational commands.
  • api.py wraps the Stable Diffusion API endpoints for model listing, options, progress and txt2img.
  • watcher.py polls Reflect notes for embedded generation triggers and appends result links back into Reflect.
  • trigger.py and trigger.ahk suggest a faster interactive flow around copied prompts and hotkeys.
  • gdrive.py uploads outputs to Google Drive and makes them publicly readable.

03 / Surface Inventory

Relevant files

  • main.py — primary CLI application.
  • api.py — Stable Diffusion API client.
  • server.py — local FastAPI image bridge on port 8000.
  • watcher.py — polling automation against Reflect daily notes.
  • trigger.py and trigger.ahk — manual/hotkey trigger path.
  • gdrive.py — Google Drive upload helper.
  • config.py — default settings and local config path resolution.
  • start.bat — Windows startup sequence for the local server and AutoHotkey trigger.

04 / Inconsistencies

What the folder suggests

  • requirements.txt lists only typer[all], rich, requests and Pillow.
  • The visible code also imports fastapi, uvicorn, and multiple Google API/auth packages, which means the requirements file is incomplete for a clean setup.
  • gdrive_credentials.json exists inside the project folder, which makes this copy operationally sensitive even though the archive only documents it.
  • __pycache__/ and .claude/settings.local.json look local/generated rather than core project source.
