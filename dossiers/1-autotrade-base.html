AUTOTRADE-BASE

File No. TS-001
Category Trading Systems
Status In Development / Private

Dossier / Present State

A local TradingView signal router built as a single Node process with an embedded dashboard, disk-based config and logs, and a TCP bridge toward MT4/MT5 execution environments.

Language Node.js / CommonJS
Bridge MQL4 EA + TCP socket
Port 80 by default
Surface Dashboard + webhook API

01 / Technical Reading

What exists today

Framework
No web framework. The service uses Node's native http module and manual routing.
Routing
Direct route handling for /, /dashboard, /webhook, /api/logs, /api/status, /api/config, destination and strategy endpoints, and /api/test.
Styling system
The dashboard UI is embedded directly inside server.js, including HTML, CSS and client-side JavaScript.
Persistence
Configuration is stored in config/config.json. Signal history is appended to logs/signals.jsonl.
Deployment model
Local Windows runtime, usually on port 80, with admin privileges or a reserved HTTP URL ACL.
Integration model
TradingView posts signals into /webhook; the server parses them and routes them either to a local file or to MT4/MT5 over TCP.
Related file
A MetaTrader Expert Advisor exists as SignalReceiver.mq4 in the project root.

02 / Notes

Observed structure

This is infrastructure more than product: one process, one dashboard, one webhook surface, one local bridge.

  • The project is effectively monolithic: core server logic, dashboard markup, styling, and client-side behavior live in one main file.
  • The dependency model is intentionally minimal. The existing lockfile does not list external npm packages.
  • The dashboard is not routed through a separate frontend build step; it is served directly by the Node process.
  • The configuration already contains real strategy and destination records, which suggests operational usage rather than a blank prototype.
  • The project appears designed for private/local operation, not public cloud deployment.

03 / Surface Inventory

Relevant files

  • server.js — Node server, dashboard UI, webhook intake, routing, and MT4/MT5 socket dispatch.
  • config/config.json — destinations and strategy mapping.
  • logs/signals.jsonl — append-only signal log.
  • SignalReceiver.mq4 — MetaTrader-side bridge component.
  • README.md — operational notes and local running instructions.

04 / Inconsistencies

What the documentation says versus what is present

  • The README mentions npm start, but there is no visible package.json in the folder.
  • The README points to docs/mt4-ea.mq4, while the available EA file is SignalReceiver.mq4 at the project root.
  • The project describes itself as zero-dependency Node, which matches the minimal lockfile, but the repository metadata is incomplete for standard npm usage.
