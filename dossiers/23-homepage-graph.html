01 / Technical Reading

What exists today

Project type
A personal archive surface disguised as a graph explorer rather than a conventional homepage.
Framework
No app framework. The visible experience is a static index.html page using three and 3d-force-graph from CDN.
Routing
No route system exists. Document nodes open static HTML files from the docs/ folder in a new tab.
Styling
All styling is embedded directly in index.html, with a dark control-panel aesthetic around the graph canvas.
Build system
A small Node utility, scripts/build-graph.cjs, compiles categories and document metadata into graph.json.
Deployment
The manual explicitly supports both GitHub Pages and Vercel as static-host targets.

02 / Notes

Observed structure

This is less a homepage than a cartographic interface for a personal document universe.

  • The real authored system is the relationship model: owner → categories → subcategories → documents → tags.
  • The site includes a practical content workflow: document HTML files, category text source, graph build step, and deploy notes.
  • Because the structure grows through more notes and docs, the most accurate state is living rather than archived.
  • The project remains lightweight; the dependency footprint is minor and the authored content carries the value.

03 / Surface Inventory
