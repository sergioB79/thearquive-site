01 / Technical Reading

What exists today

Project type
A trading publication system rather than a plain dashboard: one ingestion pipeline, one cached public lane, and one premium conversion surface.
Framework
Next.js 14 with the App Router, React 18, and TypeScript.
Routing
The archive includes route surfaces for /, /premium, /sample, /subscribe, /legal, and API routes such as /api/latest and /api/webhook/ingest.
Styling
Styling is handled through the app-level CSS in app/globals.css, with the visible product leaning into premium editorial/trading presentation.
Deployment
The intended deployment target is Vercel, with Vercel Cron triggering the ingestion endpoint on schedule.
Storage model
The project combines Google Drive as source input, Supabase for historical persistence, and Upstash Redis for fast latest-snapshot reads.

02 / Notes

Observed structure

This is a content-to-product pipeline: the market brief becomes a site, not just a file.

  • The ingestion path is serious and production-shaped: secret-protected webhook, Drive pull, markdown parse, Zod validation, database persistence, diffing, and Redis cache.
  • The copied folder contains both heavy node_modules and a large .next build cache, which explains most of the size.
  • The homepage already exposes a coherent freemium model: free ranking preview and CTA into premium content.
  • The project summary and TODO material show an active roadmap rather than a frozen artifact.
