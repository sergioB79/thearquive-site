01 / Technical Reading

What exists today

Project type
An interactive browser-based art and atmosphere experiment.
Framework
No framework is used; the project is a static front-end experience with custom assets, vanilla JavaScript, and browser APIs.
Interaction model
The core scene combines looping video, clickable bubbles that reveal messages, ambient sound cues, and a visible global click counter updated through Firebase Realtime Database.
Styling system
The visual layer is fully hand-authored in CSS, including custom fonts, glitch overlays, floating bubbles, counters, banners, and fixed signature/disclaimer elements.
Behavior
The JavaScript handles bubble spawning, video switching, glitch triggers, audio playback, disclaimer state, and Firebase transaction updates for clicks.
Weight note
Most of the folder size comes from media assets — videos, sounds, fonts, and icons — which are essential to the piece rather than accidental residue.

02 / Notes

Observed structure

This is not a site with a feature. It is a mood machine with a counter attached.

  • The piece depends on accumulation of small effects: sound, drift, glitch, hidden messages, and click ritual.
  • The Firebase counter is conceptually important because it turns anonymous interaction into a live trace.
  • It belongs in the archive as an experiment in atmosphere and participation, not as a utility product.
