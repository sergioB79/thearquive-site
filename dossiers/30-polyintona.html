A modern mouse-driven musical instrument inspired by Music Mouse, built as a JUCE desktop app where XY input shapes pitch, filter, tempo, scale, chord mode, voice count, and arpeggiation behavior.

01 / Technical Reading

What exists today

Project type
A desktop musical performance instrument, not a web product.
Framework
JUCE-based C++ application built with CMake.
Routing
No routing layer exists; the product is a native window with controls and an interactive pad.
Input model
An XY pad drives sound parameters in real time while additional controls define musical system behavior.
Build shape
The README points to a standard local JUCE/CMake workflow on Windows.
State
The current MVP already defines pitch/filter interaction, scales, chords, rhythm modes, voices, and arp modes.

02 / Notes

Observed structure

This is a software instrument with a clear performance metaphor, not just a synthesis experiment.

  • The interaction idea is legible immediately: move the mouse, bend the harmonic system.
  • The project seems serious enough to be playable, even if still MVP-shaped.
  • The large weight likely comes from JUCE/external build artifacts rather than authored source volume.
