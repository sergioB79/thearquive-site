A mystic palm-reading web app with a static upload interface in the public folder and a small Express backend that forwards hand images to Google’s Gemini vision endpoint for brief or detailed readings.

01 / Technical Reading

What exists today

Project type
An AI-powered novelty reading app framed as a mystical consumer experience.
Framework
The frontend is plain static HTML, CSS, and JavaScript under public, while the backend is a small Node and Express server under backend.
Routing
The public experience is a single page, and the server currently exposes one main POST /analyze route plus a root handler that serves the interface.
AI layer
The backend sends two uploaded hand images to Gemini using a prompt that changes between a short teaser reading and a more expansive paid-style reading.
Styling system
The UI uses Tailwind via CDN and a separate custom stylesheet, so the visual layer stays lightweight and static-host friendly.
Deployment note
The repo includes vercel.json and is clearly shaped for Vercel deployment, with the Express app exported for serverless execution.

02 / Notes

Observed structure

Technically simple, but commercially legible: upload, tease, upsell, reveal.

  • The strongest design move is the funnel logic from free brief reading to paid detailed reading.
  • The product depends on prompt framing and theatrical tone more than on heavy application architecture.
  • The implementation is compact enough to move quickly, which suits this kind of experiment.
