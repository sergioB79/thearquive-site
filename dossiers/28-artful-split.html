01 / Technical Reading

What exists today

Project type
A more productized browser variant of the image-splitting idea, with gallery, presets, toast UX, and hosted-function integration.
Framework
Vite, React 18, TypeScript, React Router, TanStack Query, and a large shadcn/Radix component footprint.
Routing
Client-side routing only, with a main / page and catch-all not-found page.
Styling
Tailwind plus component-library primitives, Framer Motion transitions, and a strong product-shell layout.
Generation path
The app calls a Supabase edge function named generate-image instead of a local Express service.
State model
The app keeps a local gallery in browser storage, making it feel closer to a real consumer tool than a one-off prototype.

02 / Notes

Observed structure

This is the hosted, UX-heavier sibling of the face-splitting concept.

  • It overlaps conceptually with `FaceFusion.AI`, but the runtime shape is different enough to justify a separate dossier.
  • The edge-function integration and local gallery make it more app-like than the earlier local prototype.
  • Most of the weight comes from `node_modules`; the authored source is comparatively small.
  • The branding and app flow suggest a stronger consumer-product angle than the raw prototype.
