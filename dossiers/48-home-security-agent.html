01 / Technical Reading

What exists today

Project type
A local security visibility tool rather than a traditional antivirus product.
Framework
The runtime is Python 3.11+ with a long-lived agent loop plus a Streamlit dashboard; supporting modules live under homeagent/.
Routing
No web framework routing exists for the core system; the dashboard uses Streamlit tabs and the public-facing documentation site is plain static HTML.
Operational model
The agent ingests selected Sysmon events, correlates behavior, scores findings, writes to SQLite, and triggers deduped alerts through Windows channels.
Documentation surface
The repo includes both a full manual and a polished static marketing/documentation site that explains scope, screenshots, and quickstart instructions.
Deployment read
This is designed for local Windows deployment with Task Scheduler, ProgramData storage, and explicit reliance on Sysmon being installed and tuned.

02 / Notes

Observed structure

The strongest choice here is honesty about scope.

  • The documentation repeatedly states that this is visibility, not blocking, which gives the whole system a serious tone.
  • The dashboard is clearly not a toy; it includes findings, telemetry, persistence lookups, allowlisting, and export flows.
  • The public site and the actual tool are both present, which makes this folder unusually complete.
