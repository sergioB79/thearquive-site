01 / Technical Reading

What exists today

Project type
An authored publishing index for trading texts and interactives.
Framework
No frontend framework is used; the site is a static HTML/CSS/JS build with a Python generator that compiles the article catalogue into index.html.
Routing
Navigation is file-based, with direct links to article pages and a Vercel rewrite rule that maps /artigos/:slug to static HTML artifacts.
Styling system
The visual language is hand-coded in a long inline or standalone stylesheet, leaning on serif typography, mono metadata, and archive-like filter controls.
Behavior
Vanilla JavaScript handles category filters, search, featured content logic, and sharing helpers.
Deployment read
vercel.json is present and clearly tuned for static hosting with simple rewrites and security headers.

02 / Notes

Observed structure

More journal index than homepage, which is exactly the point.

  • The code is simple, but the information architecture is already dense and intentional.
  • The Python build script is the central organizing mechanism, not an afterthought.
  • The site treats articles as a durable corpus rather than one-off posts.
