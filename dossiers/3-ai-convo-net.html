02 / Notes

Why the folder is large

The size appears to come from corpus data and generated indexes, not from local AI model weights.

  • Total observed size is roughly 6.24 GB.
  • input/ accounts for about 5.48 GB, with input/OpenAI alone around 5.23 GB.
  • pipeline/ is much smaller at around 0.33 GB.
  • Inside pipeline/, the heaviest areas are classified, batches and chroma_db, which read as generated processing outputs and local index storage.
  • No clear signs of local model weights were found: no .safetensors, .gguf, .onnx, .pt or similar model packages are visible.
  • The visible .bin files are too few to explain the size and do not, by themselves, indicate a local model runtime.

03 / Surface Inventory

Relevant files and layers

  • pipeline/extract_topics.py — post-processing script that reads all_classified.json and writes a reduced topic extract.
  • pipeline/extract_topics_v2.py — similar extractor, but preserving a chroma_id compatible with the indexing layer.
  • pipeline/all_classified.json — large consolidated classification file.
  • pipeline/01_parsed/*.json — parsed source conversations.
  • pipeline/classified/ and pipeline/batches/ — batch-level classification artifacts.
  • pipeline/chroma_db/ — local vector database for search/retrieval.

04 / Extension Read

Observed file profile

  • The folder contains many .png and .jpg files, suggesting rich conversation exports or media attachments.
  • There are also .json, .wav, .mp4, .csv, .zip and .pdf files, which points to a broad export/archive corpus.
  • The code surface is comparatively small: a handful of Python scripts and a compact top-level document set.

05 / Classification

What looks essential versus regenerable

  • Likely source-of-truth: the raw exports under input/, the top-level project docs, and the Python scripts.
  • Likely generated outputs: pipeline/batches, pipeline/classified, pipeline/all_classified.json, and pipeline/chroma_db.
  • Important caution: generated does not mean disposable. If the pipeline is not fully reproducible yet, these outputs may still be operationally necessary.
