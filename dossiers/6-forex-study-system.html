01 / Technical Reading

What exists today

Project type
A real web application prototype, not just a plan document: the folder contains a functioning dashboard flow, dynamic asset routes and server-side API handlers.
Framework
next@16.2.1 with react@19.2.4, react-dom@19.2.4 and the App Router under src/app.
Routing
/ redirects to /dashboard; dynamic asset dossiers live at /asset/[id]; API routes exist at /api/market/[symbol] and /api/news/[symbol].
Styling
Tailwind CSS v4 via @import "tailwindcss" in globals.css, with a custom dark theme token set and bespoke TipTap editor styling.
State model
Active assets, studies and day notes are persisted in localStorage; no live database is wired yet despite the presence of @supabase/supabase-js in dependencies.
Editor surface
TipTap is used for rich-text note and study editing through @tiptap/react and @tiptap/starter-kit.
External data
Server routes call Twelve Data for price/quote data and Alpha Vantage for ranked news sentiment, with cache revalidation set in the fetch layer.
Deployment shape
This is Vercel-friendly Next.js, not GitHub Pages-friendly static output, because the app depends on runtime API routes and environment keys.

02 / Notes

Observed structure

The concept is serious. The current build is already usable as a local research instrument, but its persistence and deployment layers are still transitional.

  • The folder contains both a planning brief and a more advanced implementation than the brief alone would suggest.
  • The dashboard is client-heavy and uses browser storage as the primary source of truth, which makes it fast locally but not yet multi-device or collaborative.
  • The per-asset page is the strongest surface: market snapshot, timeframe study stacks and filtered news are already unified in one dossier view.
  • The dependency list still reflects intended future scope: Supabase is installed, but the visible source currently persists to the browser instead of a backend.
  • A /settings link exists in the dashboard UI, but no matching route is present in the inspected file set.

04 / Weight & Residue

Why the folder is large

  • node_modules occupies roughly 466 MB.
  • .next occupies roughly 245 MB.
  • The actual authored source under src is tiny by comparison at roughly 0.08 MB.
  • The copy also contains its own nested .git folder, plus build metadata such as tsconfig.tsbuildinfo.
  • So the folder is large mostly because this is a working local clone with install and build artifacts, not because the handwritten codebase itself is massive.

05 / Deployment Reading

Operational notes

  • Live market data requires TWELVE_DATA_KEY in the runtime environment.
  • News sentiment requires ALPHA_VANTAGE_KEY in the runtime environment.
  • No vercel.json or custom deployment manifest is present; the setup relies on standard Next.js conventions.
  • next.config.ts only sets allowedDevOrigins, which suggests a local/dev networking adjustment rather than a production policy layer.
